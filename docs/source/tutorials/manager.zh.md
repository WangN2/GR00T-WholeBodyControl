# 接口管理器（一体化）

使用热键在运行时动态切换键盘、手柄和 ZMQ 输入接口（`--input-type manager`）。管理器同时拥有所有接口，并将控制权委托给当前激活的接口。你可以随时切换，无需重启程序。

```{admonition} 前置条件
:class: note
请先完成[快速开始](../getting_started/quickstart.md)，确保 sim2sim 循环已运行。
```

```{admonition} 紧急停止
:class: danger
随时按 **`O`** 即可立即停止控制并退出 —— 无论当前激活的是哪个接口，包括手柄被选中时也是如此。请始终保持一只手在键盘附近准备按 **`O`**。
```

## 启动

**Sim2Sim（MuJoCo）：**

```bash
# 终端 1 — MuJoCo 仿真器（从仓库根目录）
source .venv_sim/bin/activate
python gear_sonic/scripts/run_sim_loop.py

# 终端 2 — C++ 部署（从 gear_sonic_deploy/）
bash deploy.sh --input-type manager sim
```

**真实机器人：**

```bash
# 在 gear_sonic_deploy/ 目录下
bash deploy.sh --input-type manager real
```

如果你计划使用 ZMQ 接口，请添加 ZMQ 参数：

```bash
bash deploy.sh --input-type manager \
  --zmq-host <publisher-ip> \
  --zmq-port 5556 \
  --zmq-topic pose \
  sim
```

## 分步指南

1. 管理器默认以 **键盘** 模式启动。
2. 按 **`]`** 启动控制系统（键盘模式）。按照[键盘教程](keyboard.md)中的说明使用机器人。
3. 要切换到手柄，按 **`Shift+2`**（输入 `@`）。终端将打印 `Switched to: GAMEPAD (safety reset triggered)`。机器人会返回到参考动作的第 0 帧，规划器被禁用。
4. 按照[手柄教程](gamepad.md)中的说明使用手柄控制。
5. 要切换到 ZMQ 串流，按 **`Shift+3`**（输入 `#`）。会触发安全重置，终端打印 `Switched to: ZMQ`。按照[ZMQ 教程](zmq.md)中的说明使用 ZMQ 控制。
6. 要随时切回键盘，按 **`Shift+1`**（输入 `!`）。
7. 按 **`O`** 从任意接口停止控制并退出。

## 切换接口

| 热键 | 接口 | 说明 |
|--------|-----------|-------|
| **Shift+1** (`!`) | 键盘 | 默认。完整的键盘控制（普通 + 规划器模式）。参见[键盘教程](keyboard.md)。 |
| **Shift+2** (`@`) | 手柄 | Unitree 无线手柄。参见[手柄教程](gamepad.md)。 |
| **Shift+3** (`#`) | ZMQ | ZMQ 串流（需要提供 `--zmq-host` 等参数）。参见[ZMQ 教程](zmq.md)。 |

```{tip}
当使用 ROS2 支持构建时，也可以通过 **Shift+4** (`$`) 使用 **ROS2** 接口。它要求规划器已加载，否则回退到键盘。ROS2 接口作为构建自定义 ROS2 控制管道的参考实现提供，可能不会获得与其他接口相同级别的更新。
```

### 切换时会发生什么

每次切换都会触发**所有**被管理接口的**安全重置**（不仅仅是你要切换到的那个）。这将会：

- 禁用规划器并返回到参考动作的第 0 帧
- 重置朝向和移动状态
- 禁用 ZMQ 串流（如果之前处于激活状态）
- 防止来自前一个接口的过时状态导致的控制不连续

安全重置确保无论你之前使用的是哪个接口，切换后总能从一个干净的状态开始。

```{note}
四个接口在启动时就被创建，并在切换过程中保持存活。它们的内部状态（例如手柄模式、ZMQ 连接）会被保留 —— 只有规划器/动作状态会被安全重置。这意味着切换回来时无需重新建立 ZMQ 连接或重新配对手柄。
```

## 全局控制

这些控制在管理器层面生效，**无论当前激活的是哪个接口**。它们会在被传递给激活接口之前被管理器拦截。

### 紧急停止

| 按键 | 动作 |
|-----|--------|
| **O** / **o** | 立即紧急停止 —— 即使当前激活的是手柄或 ZMQ 接口也有效 |
| **F** / **f** | 报告电机温度（TTS 语音播报） |

### 柔顺度控制

这些全局调整手部柔顺度和抓取参数。它们在遥操作（ZMQ / ROS2）期间特别有用，因为此时机器人的手会与物体交互。

| 按键 | 动作 |
|-----|--------|
| **G** / **H** | 增加 / 减小左手柔顺度 0.1 |
| **B** / **V** | 增加 / 减小右手柔顺度 0.1 |
| **X** / **C** | 增加 / 减小最大手部闭合比例 0.1 |

```{note}
柔顺度控制是全局的 —— 无论当前激活的是哪个接口，它们都会影响机器人的手。这让你可以在手柄或 ZMQ 控制机器人移动的同时，从键盘调整柔顺度。
```

## 接口专属控制

一旦某个接口被激活，它的所有常规控制都会按照各自教程中的说明工作：

- **键盘** (`!`)：[键盘教程](keyboard.md)中的所有按键 —— T/R/P/N 控制动作，ENTER 切换规划器，WASD 移动等。
- **手柄** (`@`)：[手柄教程](gamepad.md)中的所有按钮 —— Start、A/B、L1/R1、摇杆等。
- **ZMQ** (`#`)：ENTER 切换串流，T/P/N/R 控制非串流状态下的参考动作。参见[ZMQ 教程](zmq.md)。

唯一的例外是：
- **`O`** 始终被管理器拦截用于紧急停止（不会传递给激活接口）。
- **`G/H/B/V/X/C`** 始终被拦截用于柔顺度控制。
- **`!/@/#/$`** 始终被拦截用于接口切换。
